## Findings fixed during validation

1. Firefox BiDi could attach but navigation failed because Portage Firefox content processes crashed under the unavailable container user-namespace sandbox. The wrapper now disables Firefox process sandboxes inside the pod while preserving the Kubernetes, NetworkPolicy, backend identity, sanitization, and final-purchase boundaries.
2. Firefox initially still showed `SEC_ERROR_UNKNOWN_ISSUER` for `browser.eyrie` because `www-client/firefox` overwrote `libnssckbi.so` after the base cert class linked it to p11-kit trust. The tool class now orders Firefox before `nest::base::certs`; the live image has `/usr/lib64/libnssckbi.so -> /usr/lib64/pkcs11/p11-kit-trust.so`.
3. The initial source-managed image used x11vnc plus noVNC and then client-side `resize=scale`, which made the Firefox surface appear enlarged compared with upstream KasmVNC. The current source changes replace that shim with source-built KasmVNC from `kasmtech/KasmVNC`, built via `nest::lib::src_repo` and `nest::lib::build`, and serve the KasmVNC web client directly from `/opt/kasmweb/share/kasmvnc/www`.
4. The wrapper preserved runtime `HOME=/root`, putting Firefox state on ephemeral root storage instead of the mounted PVC. It now defaults `HOME` to `/home/kasm-user` unless `FIREFOX_HOME` is explicitly set; the live pod creates the profile under `/home/kasm-user/.mozilla/firefox/nest-secure-browser`.
5. Firefox could open smaller than the VNC framebuffer without a desktop window manager. The wrapper now derives `FIREFOX_WIDTH`/`FIREFOX_HEIGHT` from `VNC_RESOLUTION`, passes them to Firefox, and uses `xdotool` to move/resize the visible Firefox window to `0,0 1365x768`.
6. The Bitwarden enterprise policy was mounted at `/usr/lib/firefox/distribution/policies.json`, while Gentoo Firefox reads `/usr/lib64/firefox/distribution/policies.json`. The workload now mounts policy at the Gentoo path.
7. The image now caches the public Bitwarden XPI at `/opt/nest/firefox/extensions/bitwarden.xpi` and exposes it through `/usr/lib64/firefox/distribution/extensions/{446900e4-71c2-419f-a6a7-df9c091e268b}.xpi`; the live profile has the extension XPI under `/home/kasm-user/.mozilla/firefox/nest-secure-browser/extensions/`.
