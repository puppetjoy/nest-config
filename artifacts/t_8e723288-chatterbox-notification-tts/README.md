# t_8e723288 Chatterbox notification TTS evidence

## Scope

Joy observed two production Chatterbox notification issues after the previous tuning rollout:

- an audible gasp/breath after the phrase “Agent request received”
- garbled spoken task ids such as `t_922b6ec2`

This source change keeps Telegram/visual notification text canonical and changes only the speech input sent to Chatterbox.

## Source change summary

`data/kubernetes/app/voice-chatterbox.yaml` updates the embedded `voice-chatterbox` server wrapper from `0.1.1` to `0.1.2`:

- adds regex-based notification text normalization to smooth hard whitespace/punctuation boundaries after `Agent request received`
- spells task ids as `task T ...` instead of sending raw `t_<hex>` to the model
- spells Agent Request ids as `agent request A R dash ...`
- ports the operational-token normalization already used by `voice-speech` for `KubeCM`, `llama-qwen`, `OpenVox`, `ROCm`, and `kubectl`
- adds `/v1/audio/normalize` for read-only verification of the speech string
- reports the normalization feature in `/health`
- returns `X-Nest-TTS-Normalized-SHA256` so callers can verify which speech string was generated without putting long text in headers

`data/kubernetes/service/voice-chatterbox.yaml` bumps `cutover_revision` to force a managed Kubernetes rollout when this source is approved and deployed.
